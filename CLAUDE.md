This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

구미시 대상 AR 기반 부동산 탐색 앱. 스마트폰 카메라로 건물을 비추면 실거래가, 건축물대장 정보, 아파트 단지 정보가 AR로 오버레이된다. 현재 **기획/조사 단계**로, 앱 코드는 아직 없으며 기술 조사 문서와 Python 데이터 조회 프로토타입으로 구성되어 있다.

## 저장소 구성

- **조사 문서 (Markdown)**: API 조사, 건물인식 기술조사, 레퍼런스 분석, 통합 보강본
- **`okgye_building_data.py`**: 옥계동 아파트/건물 데이터를 공공 API 3종으로 조회하는 Python 프로토타입
- **`260402_주제선정_회의자료_강형준.pdf`**: 프로젝트 킥오프 회의 자료
- **`20230020_강형준_과제4.hwp`**: 과제 제안서 원본

## 프로토타입 실행 방법

```bash
pip install requests
python okgye_building_data.py
# 실행 시 data.go.kr API 키 입력 프롬프트 표시
```

https://www.data.go.kr 에서 API 키 발급 필요 (아래 3개 서비스 활용신청, 인증키 공유):
- 건축물대장정보
- 아파트 매매 실거래가
- 공동주택 단지 목록

## 주요 코드 및 상수

- 구미시 시군구코드: `47190`
- 옥계동 법정동코드: `12800` (전체: `4719012800`)
- 구미시 좌표 범위: 경도 128.20~128.50°E, 위도 36.05~36.25°N

## 설계된 아키텍처

**데이터 파이프라인** (조사 문서에 상세 기술):
1. 사용자 위치 + 카메라 방향
2. V-World 역지오코딩 (좌표 → 주소)
3. V-World 2D Data API (주변 건물 폴리곤, 레이어 `LT_C_BLDGINFO`)
4. 건축물대장 API (건물 물리적 정보)
5. 공동주택 단지 API (단지/세대 상세)
6. 실거래가 API (시세 정보)
7. AR 오버레이 렌더링

**건물 인식 방식**: 이미지 분류가 아닌 하이브리드 공간 매칭 (GPS + 방위각 + 건물 폴리곤 기하). 동 번호 OCR(101동, 102동)과 창문 패턴 감지를 보조 수단으로 활용. 층 식별은 건물 높이 / 총 층수로 층별 밴드를 산출한다.

## 공공데이터 API

두 가지 주요 데이터 포털:
- **data.go.kr** (공공데이터포털): 건축물대장 (11개 하위 서비스), 아파트 거래 (매매/전월세), 공동주택 단지 정보
- **vworld.kr** (브이월드): 지오코딩, 2D 건물 폴리곤 데이터 (WFS/WMS)

API 응답 형식은 엔드포인트에 따라 JSON 또는 XML로 다르다. 프로토타입은 두 형식 모두 처리한다. 성공 코드도 API마다 다름 (`"00"` 또는 `"000"`).
