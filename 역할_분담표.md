### R2. 기술 리드 / AR 핵심 개발

> **영역**: 프론트엔드 핵심 + 전체 기술 아키텍처  
> **핵심 키워드**: 앱 아키텍처, ARCore, 건물 인식, 기술 가이드

#### 담당 업무

**앱 아키텍처 설계 및 프로젝트 세팅**
- 앱 프레임워크 결정 및 프로젝트 초기 세팅 (Android Kotlin / Flutter / Unity)
- 전체 시스템 아키텍처 설계 (프론트엔드 ↔ 백엔드 통신 구조)
- API 명세서 작성 (엄태원과 협의하여 엔드포인트 정의)
- 프론트엔드 폴더 구조, 코드 컨벤션 세팅
- R4(권희성)·R5(나웅주)가 작업할 수 있도록 기본 화면 틀/라우팅 구성

**ARCore Geospatial API 연동**
- ARCore SDK 세팅 및 Geospatial API 초기화
- VPS 기반 정밀 위치/방향 파악
- Geospatial Anchor로 AR 콘텐츠 고정
- Rooftop Anchor로 동별 라벨 배치
- Streetscape Geometry 건물 3D 메시 수신 및 Hit-Test
- Geospatial Depth API (65m 거리 깊이, 오클루전)
- VPS 미지원 지역에서 GPS+방위각 **폴백 로직** 구현

**건물 인식 알고리즘 구현** (통합문서 섹션 11)
- GPS + 방위각으로 주변 건물 후보 추출 (반경 150~300m)
- 카메라 FOV 내 건물 필터링 (bearing 계산)
- 건물 메시 중심좌표 ↔ GIS 건물 DB 좌표 매칭 (Haversine)
- 신뢰도 점수 계산 (거리 + 시야 일치도 + 메시 일치도 + OCR)
- 확신 낮을 시 후보 2~3개 제시 로직

**동번호 OCR**
- 온디바이스 OCR (ML Kit) 세팅
- 카메라 프레임에서 아파트 외벽 동번호(101, 102 등) 인식
- OCR 결과 + 공간 매칭 결과 통합

**층 추정 로직** (Phase 3)
- 건물 총 높이 / 총 층수로 층별 밴드 계산
- AR hit-test 히트포인트 Y좌표로 층 추정
- 1층 필로티/상가 오프셋 보정

**AR 태그 렌더링**
- AR 화면에 건물별 정보 태그 (단지명, 거래가, 편의시설 점수)
- 태그 탭 → R4(권희성)가 만든 상세 화면으로 연결
- 거리에 따른 태그 크기/투명도 조절

**R4·R5 팀원 기술 가이드**
- 권희성·나웅주에게 프레임워크 사용법 가이드
- 화면 개발 시 막히는 부분 기술 지원
- 코드 리뷰

#### 핵심 산출물

### R3. 백엔드 통합 (API + DB + 점수화)

> **영역**: 백엔드 전체  
> **핵심 키워드**: FastAPI, 공공데이터 연동, PostgreSQL+PostGIS, 편의시설 점수화

#### 담당 업무

**API 서버 구축**
- FastAPI (Python) 기반 REST API 서버
- 강형준(R2)이 작성한 API 명세에 따라 엔드포인트 구현
- 주요 API 엔드포인트:
  - `GET /buildings?lat=&lon=&radius=` → 주변 건물 목록
  - `GET /buildings/{id}/trades` → 실거래 내역
  - `GET /buildings/{id}/info` → 건축물대장 정보
  - `GET /buildings/{id}/livability` → 편의시설 점수
  - `GET /complexes/{kaptCode}` → 단지 상세 정보
- 에러 핸들링, 로깅, CORS 설정

**공공데이터 API 연동** (통합문서 섹션 4)

| 연동 대상 | 용도 |
|-----------|------|
| 아파트 매매 실거래가 API | 핵심 가격 데이터 |
| 아파트 전월세 실거래가 API | 전세/월세 |
| 건축물대장 표제부/층별개요 | 건물 상세 (층수, 면적, 구조) |
| 공동주택 단지 목록 | kaptCode 획득 |
| 공동주택 기본정보 | 단지 메타데이터 |
| 한국부동산원 단지 식별정보 | 동정보, 단지명 이력 |
| Vworld 역지오코딩 / 2D 데이터 / WFS | 좌표↔주소, 건물 폴리곤 |

- XML/JSON 응답 파싱 및 정규화
- 페이징 처리 (numOfRows, pageNo)

**DB 설계 및 구축**
- PostgreSQL + PostGIS 설치 및 세팅
- 핵심 테이블:
  - `apt_complex_master` — 아파트 단지 마스터
  - `apt_building_master` — 아파트 동 마스터 (폴리곤 포함)
  - `apt_trade_history` — 거래 이력
  - `living_infra_gumi` — 구미시 생활 인프라 (편의시설)
  - `building_livability_score` — 건물별 편의시설 점수 캐시
- 공간 인덱스 (GIST) 설정
- 여창훈(R1)이 수집한 구미시 데이터 DB 적재

**구미시 건물 마스터 DB 구축**
- GIS건물통합정보 → PostGIS 적재
- 단지/동 정규화 (공동주택 목록 + 한국부동산원 동정보 매칭)
- 구미시 법정동코드 매핑

**편의시설 점수화 엔진** (통합문서 섹션 10)
- Kakao Local API 연동 (카테고리 검색: 병원/약국/편의점/마트/은행/학교/지하철/문화시설)
- 여창훈(R1)이 수집한 구미시 데이터 활용 (버스 정류장, CCTV, 어린이집 등)
- Haversine 거리 계산
- 거리별 점수 변환 (300m=100점 ~ 1,600m+=15점)
- 밀도 보너스 계산
- 카테고리별 점수 + 가중치 프리셋(기본/육아/직장인/시니어) → 종합 점수 + 등급(S~F)
- 점수 결과 캐싱 (주 1회 재계산)

**PostGIS 공간 쿼리**
- 반경 N미터 내 건물/편의시설 조회 (`ST_DWithin`)
- 건물-좌표 간 거리 계산 (`ST_Distance`)
- BBOX 영역 검색

**캐싱**
- 실거래가: 일 1회 갱신
- 건물/단지 정보: 주 1회 갱신
- 편의시설 점수: 주 1회 재계산
- Redis 또는 인메모리 캐시

### R4. 프론트엔드 - 앱 화면 / 상세 UI

> **영역**: 프론트엔드 (모바일 앱 화면)  
> **핵심 키워드**: 드릴다운 UI, 건물 상세 카드, 층별 보기, 필터, 그래프  
> **가이드**: 강형준(R2)이 프레임워크 세팅 및 기술 지원

#### 담당 업무

**건물 상세 카드 (Level 2)**
- AR 태그 탭 시 열리는 건물 상세 화면
- 표시 항목:
  - 단지명, 동번호
  - 최근 거래 5건 리스트
  - 전용면적 타입별 가격
  - 건축년도, 세대수, 주차대수, 승강기
  - 사용승인일, 구조, 난방방식

**층별 보기 (Level 3)**
- 세로형 층 바 UI
- 저층/중층/고층 구분 표시
- 최근 거래가 있는 층 하이라이트
- 특정 층 탭 → 해당 층 거래 상세 (금액, 면적, 계약일)

**거래 이력 화면**
- 해당 건물의 거래 내역 리스트 (최신순)
- 거래 항목: 거래일, 금액, 전용면적, 층, 거래유형(매매/전월세)

**필터링 / 검색**
- 면적 범위 필터 (슬라이더)
- 가격 범위 필터 (슬라이더)
- 건축년도 필터
- 거래 유형 필터 (매매/전세/월세)
- 편의시설 등급 필터 ("A등급 이상만 표시")

**가격 변동 그래프**
- 특정 건물/단지의 시세 추이 꺾은선 그래프
- 지역 평균 대비 비교선
- 면적별 가격 추이

**AR 뷰 ↔ 지도 뷰 전환**

- 전환 시 현재 보던 건물 컨텍스트 유지

### R5. 프론트엔드 - 지도 / 편의시설 화면

> **영역**: 프론트엔드 (모바일 앱 화면)  
> **핵심 키워드**: 지도 SDK, 마커, 편의시설 레이어, 점수 화면, 비교표, 디자인  
> **가이드**: 강형준(R2)이 프레임워크 세팅 및 기술 지원

#### 담당 업무

**지도 뷰 개발**
- Google Maps SDK 또는 Kakao Map SDK 연동
- 건물 마커 표시 (단지별 대표 가격 + 편의시설 등급 색상)
- 편의시설 레이어 토글:
  - 병원/약국 아이콘
  - 학교/어린이집 아이콘
  - 편의점/마트 아이콘
  - 버스 정류장 아이콘
  - CCTV 아이콘
- 건물-편의시설 간 거리선(distance line) 시각화
- 마커 탭 → R4(권희성)가 만든 상세 화면으로 연결

**편의시설 점수 상세 화면 (Level 4)** (통합문서 섹션 10.7)
- 종합 점수 + 등급 표시 (S~F, 프로그레스 바)
- 카테고리별 점수 바 차트 (의료/교육/편의/교통/안전/여가)
- 카테고리별 최근접 시설 목록:
  - 시설명
  - 거리 (m)
  - 도보 시간 (분)
  - 반경 내 시설 수
- 가중치 프리셋 전환 버튼 (기본 균형 / 육아 가족 / 직장인 / 시니어)
- 커스텀 가중치 슬라이더 (선택)

**건물 비교 기능**
- 2~3개 건물의 가격 + 편의시설 점수 나란히 비교표
- 카테고리별 비교 (의료 95 vs 72, 교통 72 vs 80 ...)

**와이어프레임 / 디자인**
- Figma로 앱 전체 화면 흐름 와이어프레임
- 색상 팔레트, 타이포그래피 기본 가이드
- AR 태그 디자인 시안
- 편의시설 등급별 색상 (S=금, A=초록, B=파랑, C=노랑, D=주황, F=빨강)

## 3. 역할 간 협업 흐름

### 데이터 흐름

```
여창훈(R1)                    엄태원(R3)
공공데이터 수집 ──(CSV/JSON)──→ DB 적재
API 키 발급 ──(키 공유)──────→ API 연동

엄태원(R3)                    강형준(R2) + 권희성(R4) + 나웅주(R5)
REST API 제공 ──(JSON 응답)──→ 앱에서 호출하여 화면 표시

강형준(R2)
├── 프로젝트 세팅 ──(기본 틀)────→ 권희성(R4): 상세 UI 개발
├── 프로젝트 세팅 ──(기본 틀)────→ 나웅주(R5): 지도/점수 화면 개발
├── AR 태그 탭 ──(이벤트)────────→ 권희성(R4): 상세 화면으로 전환
├── AR 태그 탭 ──(이벤트)────────→ 나웅주(R5): 점수 화면으로 전환
└── 코드 리뷰 ──(피드백)─────────→ 권희성(R4) + 나웅주(R5)

나웅주(R5)
└── 와이어프레임 ──(디자인)──────→ 권희성(R4): UI 개발 시 참고
```

### 핵심 협업 포인트

| 협업 | 관련 역할 | 내용 |
|------|-----------|------|
| **API 명세 합의** | R2 + R3 | 프론트가 백엔드에 요청할 엔드포인트 정의 |
| **건물 데이터 포맷** | R2 + R3 | AR 건물 인식에 필요한 폴리곤/좌표/메타데이터 |
| **상세 화면 연결** | R2 + R4 | AR 태그 탭 → 건물 상세 카드 전환 인터페이스 |
| **점수 화면 연결** | R2 + R5 | AR 태그 탭 → 편의시설 점수 화면 전환 |
| **점수 API 연동** | R3 + R5 | 점수 API 응답 형식 → 화면 표시 |
| **디자인 공유** | R5 → R4 | 나웅주 와이어프레임 → 권희성 UI 개발 참고 |
| **데이터 전달** | R1 → R3 | 여창훈 수집 데이터 → 엄태원 DB 적재 |
| **기술 가이드** | R2 → R4, R5 | 강형준이 권희성·나웅주 기술 지원 |
